# 培训互动活动库

## 破冰活动（开场使用）

### 1. 自我介绍接龙
**时长**：15-20分钟  
**适用人数**：10-30人  
**目标**：帮助学员相互认识，营造轻松氛围

**步骤**：
1. 每人用1分钟介绍自己：姓名、部门、一个有趣的事实
2. 下一位学员要先复述上一位的信息，再介绍自己
3. 讲师可以设定主题，如"我的管理困境"、"我的学习期待"

### 2. 两个真相一个谎言
**时长**：10-15分钟  
**适用人数**：10-20人  
**目标**：活跃气氛，增进了解

**步骤**：
1. 每人说三句关于自己的话，其中两句是真的，一句是假的
2. 其他学员猜哪句是假的
3. 揭晓答案并简单解释

### 3. 期望墙
**时长**：10分钟  
**适用人数**：不限  
**目标**：了解学员期望，调整课程重点

**步骤**：
1. 发放便签纸，每人写下对课程的1-2个期望
2. 贴在白板上，讲师分类整理
3. 讲师回应哪些期望会在课程中满足

## 小组讨论活动

### 1. 头脑风暴
**时长**：15-20分钟  
**适用场景**：收集想法、激发创意

**步骤**：
1. 提出开放性问题（如"如何提升团队执行力？"）
2. 小组内快速列出所有想法，不评判
3. 整理归类，选出最有价值的3-5个
4. 各组分享，讲师总结

**引导要点**：
- 强调"数量优先于质量"，鼓励大胆发言
- 禁止批评和否定
- 设定时间限制，保持节奏

### 2. 世界咖啡
**时长**：30-40分钟  
**适用场景**：深度讨论多个主题

**步骤**：
1. 设定3-4个讨论主题，每桌一个主题
2. 学员分组，每组在一个桌子讨论10分钟
3. 轮换到下一个桌子，继续讨论新主题
4. 每桌有一位"桌主"保持不动，向新来的组员介绍前面的讨论
5. 最后回到起始桌，总结讨论成果

### 3. SWOT分析
**时长**：20-30分钟  
**适用场景**：分析问题、评估方案

**步骤**：
1. 给出分析对象（如某个管理策略、项目方案）
2. 小组讨论，填写SWOT矩阵（优势、劣势、机会、威胁）
3. 各组展示分析结果
4. 讲师点评和补充

## 案例分析活动

### 1. 案例诊断
**时长**：20-30分钟  
**适用场景**：问题分析、原因诊断

**步骤**：
1. 提供一个真实案例（如项目失败、沟通冲突）
2. 小组讨论：问题是什么？根本原因是什么？
3. 使用工具（如鱼骨图、5Why分析）进行诊断
4. 各组分享诊断结果，讲师点评

**案例设计要点**：
- 案例要贴近学员的工作场景
- 信息要充分但不冗余
- 问题要有一定复杂度，避免答案过于明显

### 2. 案例对比
**时长**：15-20分钟  
**适用场景**：方法对比、最佳实践学习

**步骤**：
1. 提供两个对比案例（一个成功、一个失败）
2. 小组讨论：两者的差异在哪里？成功的关键因素是什么？
3. 提炼可复用的经验和教训
4. 各组分享，讲师总结

### 3. 案例续写
**时长**：20-25分钟  
**适用场景**：方案设计、决策训练

**步骤**：
1. 提供案例的前半部分，描述情境和问题
2. 小组讨论：如果你是主人公，你会怎么做？
3. 设计解决方案，说明理由
4. 各组分享方案，讲师揭晓案例的真实结果并点评

## 角色扮演活动

### 1. 情景模拟
**时长**：30-40分钟  
**适用场景**：沟通技巧、冲突处理、谈判技巧

**步骤**：
1. 设定场景（如向上汇报、跨部门沟通、绩效面谈）
2. 分配角色（如管理者、下属、客户）
3. 学员按角色进行模拟对话（5-10分钟）
4. 观察者记录要点，提供反馈
5. 角色互换，再次演练
6. 讲师点评，总结要点

**引导要点**：
- 提供详细的角色卡，说明角色的背景、目标、立场
- 鼓励学员投入角色，真实表现
- 反馈要具体，指出做得好的地方和改进空间

### 2. 鱼缸讨论
**时长**：20-30分钟  
**适用场景**：观摩学习、技巧演示

**步骤**：
1. 选3-4位学员坐在中间（鱼缸内）进行讨论或角色扮演
2. 其他学员围坐在外圈观察
3. 讨论结束后，外圈学员提供反馈和建议
4. 可以轮换鱼缸内的学员，让更多人参与

## 实操演练活动

### 1. 工具实操
**时长**：20-30分钟  
**适用场景**：工具使用、模板填写

**步骤**：
1. 讲师演示工具的使用方法（如OKR模板、RACI矩阵）
2. 学员使用自己的实际工作场景填写工具
3. 小组内互相检查和反馈
4. 讲师抽查几份进行点评

**引导要点**：
- 提供清晰的填写示例
- 巡场答疑，及时解决学员的困惑
- 点评时要指出常见错误和改进方向

### 2. 技能闯关
**时长**：30-40分钟  
**适用场景**：技能训练、综合应用

**步骤**：
1. 设计3-4个难度递增的任务关卡
2. 学员按顺序完成任务，讲师或助教检查
3. 通过后进入下一关，未通过则重新练习
4. 全部通过后获得"通关证书"

**适用技能**：
- Excel操作、PPT制作等软件技能
- 数据分析、报告撰写等专业技能

## 游戏化活动

### 1. 知识竞赛
**时长**：20-30分钟  
**适用场景**：知识巩固、活跃气氛

**步骤**：
1. 将学员分成若干小组
2. 讲师提问，小组抢答或轮流作答
3. 答对得分，答错不扣分或扣少量分
4. 最后统计得分，给予奖励

**题目设计**：
- 选择题、判断题为主，便于快速作答
- 难度适中，避免过难或过简单
- 可以设置"风险题"，答对得双倍分，答错扣分

### 2. 拼图游戏
**时长**：15-20分钟  
**适用场景**：团队协作、流程理解

**步骤**：
1. 将一个完整的流程图或知识框架打印成拼图
2. 小组合作，在规定时间内完成拼图
3. 完成后讲解拼图内容，检验理解程度
4. 最快完成且讲解正确的小组获胜

### 3. 角色卡牌
**时长**：20-30分钟  
**适用场景**：理解不同角色的视角

**步骤**：
1. 准备不同角色的卡牌（如产品经理、技术负责人、销售经理）
2. 每人抽取一张卡牌，站在该角色立场讨论问题
3. 讨论结束后，分享从不同角色视角看到的差异
4. 讲师总结跨角色协作的要点

## 反思总结活动

### 1. 学习日志
**时长**：10-15分钟  
**适用场景**：每个模块结束后

**步骤**：
1. 发放学习日志模板（今天学到了什么？有什么疑问？如何应用？）
2. 学员独立填写5-10分钟
3. 自愿分享或小组内分享
4. 讲师收集共性问题，集中答疑

### 2. 行动计划
**时长**：15-20分钟  
**适用场景**：课程结束前

**步骤**：
1. 发放行动计划表（我要改进什么？具体行动是什么？时间节点？）
2. 学员独立填写
3. 小组内分享，相互监督
4. 可选：培训后1个月进行跟进检查

### 3. 一分钟总结
**时长**：5-10分钟  
**适用场景**：快速回顾

**步骤**：
1. 每位学员用一分钟总结今天最大的收获
2. 可以采用"接龙"形式，每人说一个不重复的收获
3. 讲师最后补充和强调

## 活动选择指南

根据不同的培训目标和场景，选择合适的互动活动：

| 培训目标 | 推荐活动 |
|---------|---------|
| 建立信任、打破隔阂 | 破冰活动、团队建设游戏 |
| 激发创意、收集想法 | 头脑风暴、世界咖啡 |
| 分析问题、诊断原因 | 案例诊断、SWOT分析 |
| 练习技能、掌握工具 | 工具实操、技能闯关 |
| 理解不同视角 | 角色扮演、角色卡牌 |
| 巩固知识、活跃气氛 | 知识竞赛、拼图游戏 |
| 促进反思、制定计划 | 学习日志、行动计划 |

## 活动设计的注意事项

1. **时间控制**：活动时间要严格控制，避免拖延影响整体进度
2. **规则清晰**：活动规则要简单明了，5分钟内能讲清楚
3. **分组合理**：小组人数控制在4-6人，确保每人都有参与机会
4. **产出明确**：每个活动要有明确的产出（讨论结果、演练反馈等）
5. **点评到位**：讲师要深度点评，提炼要点，而非简单总结
6. **灵活调整**：根据现场情况灵活调整活动时间和形式
